Documentação da API de Gerenciamento de Produtos e Usuários

Esta API permite gerenciar produtos e usuários, permitindo a criação, leitura, atualização e exclusão de ambos. A API também permite o gerenciamento de permissões de administradores para usuários.

1. Produtos

1.1 Criar Produto

POST /produtos

Cria um novo produto no sistema.

Exemplo de Requisição

POST /produtos
Content-Type: application/json

{
    "nome": "Produto Exemplo",
    "preco": 29.99
}

Exemplo de Resposta

{
    "id": 1,
    "nome": "Produto Exemplo",
    "preco": 29.99
}

1.2 Listar Produtos

GET /produtos

Retorna todos os produtos no sistema.

Exemplo de Resposta

[
    {
        "id": 1,
        "nome": "Produto Exemplo",
        "preco": 29.99
    },
    {
        "id": 2,
        "nome": "Outro Produto",
        "preco": 49.99
    }
]

1.3 Editar Produto

PUT /produtos/{id}

Atualiza as informações de um produto existente.

Exemplo de Requisição

PUT /produtos/1
Content-Type: application/json

{
    "nome": "Produto Atualizado",
    "preco": 39.99
}

Exemplo de Resposta

{
    "id": 1,
    "nome": "Produto Atualizado",
    "preco": 39.99
}

1.4 Deletar Produto

DELETE /produtos/{id}

Remove um produto do sistema.

Exemplo de Requisição

DELETE /produtos/1

Exemplo de Resposta

{
    "mensagem": "Produto excluído com sucesso"
}

2. Usuários

2.1 Criar Usuário

POST /usuarios

Criar um novo usuário no sistema, com opção de administrador.

Exemplo de Requisição

POST /usuarios
Content-Type: application/json

{
    "nome": "João Silva",
    "email": "joao@example.com",
    "senha": "senha123",
    "telefone": "1234567890",
    "administrador": true
}

Exemplo de Resposta

{
    "id": 1,
    "nome": "João Silva",
    "email": "joao@example.com",
    "telefone": "1234567890",
    "administrador": true
}

2.2 Listar Usuários

GET /usuarios

Retorna todos os usuários no sistema.

Exemplo de Resposta

[
    {
        "id": 1,
        "nome": "João Silva",
        "email": "joao@example.com",
        "telefone": "1234567890",
        "administrador": true
    },
    {
        "id": 2,
        "nome": "Maria Oliveira",
        "email": "maria@example.com",
        "telefone": "9876543210",
        "administrador": false
    }
]

2.3 Editar Usuário

PUT /usuarios/{id}

Atualiza as informações de um usuário existente, incluindo o campo de administrador.

Exemplo de Requisição

PUT /usuarios/1
Content-Type: application/json

{
    "nome": "João Silva Atualizado",
    "email": "joao@novoemail.com",
    "senha": "novaSenha123",
    "telefone": "1234567890",
    "administrador": false
}

Exemplo de Resposta

{
    "id": 1,
    "nome": "João Silva Atualizado",
    "email": "joao@novoemail.com",
    "telefone": "1234567890",
    "administrador": false
}

2.4 Deletar Usuário

DELETE /usuarios/{id}

Remove um usuário do sistema.

Exemplo de Requisição

DELETE /usuarios/1

Exemplo de Resposta

{
    "mensagem": "Usuário excluído com sucesso"
}

3. Login

3.1 Realizar Login

POST /login

Permite que um usuário realize login fornecendo seu email e senha.

Exemplo de Requisição

POST /login
Content-Type: application/json

{
    "email": "joao@example.com",
    "senha": "senha123"
}

Exemplo de Resposta

{
    "success": true,
    "usuario": {
        "id": 1,
        "nome": "João Silva",
        "email": "joao@example.com",
        "telefone": "1234567890",
        "administrador": true
    }
}

4. Interfaces Web

A API foi projetada para funcionar com interfaces web, permitindo que usuários e administradores interajam diretamente com a aplicação no navegador.

4.1 Página do Administrador

A página de administrador permite:

  • Criar, editar e excluir produtos;
  • Criar, editar e excluir usuários;
  • Visualizar todos os produtos e usuários registrados.

Além disso, o administrador pode marcar usuários como administradores durante a criação ou edição.

4.2 Página do Usuário Comum

A página de usuário comum permite:

  • Visualizar todos os produtos disponíveis para compra;
  • Simular a compra de produtos (a compra é fictícia e não envolve transações reais).
